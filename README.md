# AI-Powered Cybersecurity Incident Response System

An enterprise-grade cybersecurity monitoring and automated incident response platform built using ServiceNow SecOps, FastAPI, and Gemini AI. The system detects security threats, performs AI-powered threat classification, integrates with VirusTotal for IP reputation checks, and automatically routes incidents to the appropriate SOC team — with zero manual intervention.

---

## Project Overview

When a security alert is submitted, the system:

1. Stores the threat in a custom ServiceNow Security Incident table
2. Business Rule triggers automatically after record creation
3. Sends threat data to the FastAPI backend via REST API
4. Gemini AI analyzes the threat in depth
5. VirusTotal API checks the source IP for malicious activity
6. AI returns threat type, severity, assignment group, analysis, and recommended action
7. ServiceNow auto-updates all fields on the security incident record
8. Critical threats auto-escalate to Investigating status
9. Email notification sent for Critical severity threats
10. SOC Dashboard reflects live threat metrics

## Setup Guide

### Prerequisites
- Python 3.10+
- ServiceNow Personal Developer Instance (PDI)
- Gemini API key from aistudio.google.com
- VirusTotal API key from virustotal.com (free account)

### 1. Clone the Repository

```bash
git clone https://github.com/SaiKrishna-art/AI-Cybersecurity-Incident-Response.git
cd AI-Cybersecurity-Incident-Response/backend
```

### 2. Create Virtual Environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables

Create a `.env` file in the backend folder:

```
GEMINI_API_KEY=your_gemini_api_key_here
VIRUSTOTAL_API_KEY=your_virustotal_api_key_here
```

### 5. Run the Server

```bash
uvicorn main:app --reload
```

Server runs at: http://127.0.0.1:8000

API docs available at: http://127.0.0.1:8000/docs

## API Usage

### POST /analyze-threat

Analyzes a security threat and returns AI classification with VirusTotal enrichment.

**Request:**
```json
{
  "threat_description": "Ransomware detected on finance workstation, files being encrypted rapidly and spreading across network shares",
  "source_ip": "185.220.101.45",
  "threat_type": "Ransomware"
}
```

**Response:**
```json
{
  "threat_type": "Ransomware",
  "severity": "Critical",
  "assignment_group": "Threat Response Team",
  "ai_analysis": "This is an active and critical ransomware incident impacting a finance department workstation with confirmed network propagation. The VirusTotal results validate the malicious nature of the threat.",
  "recommended_action": "Immediately isolate the affected workstation and any compromised network shares to prevent further spread. Activate the incident response plan to contain, eradicate, and recover from the ransomware attack.",
  "virustotal_result": "Malicious reports: 17, Suspicious: 2"
}
```

## ServiceNow Configuration

### Custom Table: u_security_incident

| Field | Type | Purpose |
|---|---|---|
| u_threat_id | String | Unique threat identifier |
| u_threat_type | String | Type of attack |
| u_source_ip | String | Attacker IP address |
| u_severity | Choice | Critical / High / Medium / Low |
| u_ai_analysis | String | Gemini AI technical analysis |
| u_recommended_action | String | AI-generated response steps |
| u_assigned_group | Reference | SOC team assignment |
| u_status | Choice | Open / Investigating / Contained / Resolved |
| u_threat_description | String | Full threat description |
| u_virustotal_result | String | IP reputation data |

### Business Rules

| Rule | Trigger | Purpose |
|---|---|---|
| AI Threat Analyzer | After Insert | Calls FastAPI, updates record with AI results |
| Critical Threat Escalation | After Insert/Update | Auto-escalates Critical threats to Investigating |

### SOC Teams
- SOC Team
- Threat Response Team
- Endpoint Security
- Network Security

## Deployment

Backend is deployed on Render (free tier).

Note: Free tier has a cold start delay of 30-50 seconds after inactivity. Subsequent requests are fast.

To deploy your own instance:
1. Fork this repository
2. Create a new Web Service on render.com
3. Set Root Directory to `backend`
4. Set Build Command: `pip install -r requirements.txt`
5. Set Start Command: `uvicorn main:app --host 0.0.0.0 --port 10000`
6. Add environment variables: `GEMINI_API_KEY` and `VIRUSTOTAL_API_KEY`
